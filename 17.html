Optimización y Rendimiento: Fuentes Web y Tipografía Profesional

1. Google Fonts de Forma Profesional

Muchos desarrolladores simplemente copian el <link> de Google Fonts, pero esto añade peticiones externas que pueden ralentizar la carga.

Mejores prácticas:

  • Limita los pesos: No cargues estilos innecesarios; cada peso es un archivo extra.
  • Auto-hospedaje (Self-hosting): Servir las fuentes desde tu propio servidor permite usar HTTP/2 y elimina la resolución DNS externa.
  • Preconectar: Si usas servidores de Google, acelera la conexión con preconnect:
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

2. Dominando @font-face y font-display

Al hospedar tus fuentes, el formato WOFF2 es el más eficiente actualmente.

@font-face {
  font-family: 'MiFuenteCustom';
  src: url('/fonts/mi-fuente.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap; /* CRÍTICO para el rendimiento */
}
El secreto: font-display: swap
Muestra una fuente del sistema mientras se descarga la personalizada, evitando el Flash of Invisible Text (FOIT).

3. Unidades Relativas: rem vs em

Debes evitar los píxeles (px) para mejorar el rendimiento y la accesibilidad.

[Image comparing rem vs em scale based on root vs parent element]
  • rem (Root EM): Basado en el tamaño de la raíz (<html>), usualmente 16px por defecto.
  • em: Basado en el tamaño del elemento padre, ideal para espaciados proporcionales.

La técnica del 62.5%

Hace que 1rem equivalga a 10px, facilitando los cálculos:

html { font-size: 62.5%; }
body { font-size: 1.6rem; } /* 16px escalable */

4. Tipografía Fluida

Usa la función clamp() para crear texto que se adapte al Viewport sin usar múltiples media queries.

h1 { font-size: clamp(2rem, 5vw + 1rem, 4rem); }

5. Optimización de Recursos (Preload)

Indica al navegador que descargue la fuente de inmediato:

<link rel="preload" href="/fonts/mi-fuente.woff2" as="font" type="font/woff2" crossorigin>

Comparativa de Formatos y Unidades

Concepto Recomendación Por qué
Formato WOFF2 30% menos peso que WOFF.
Unidad rem Accesibilidad y escalabilidad.
Carga font-display: swap Evita textos invisibles.
Cantidad Máx. 2 familias Evita latencia innecesaria.
