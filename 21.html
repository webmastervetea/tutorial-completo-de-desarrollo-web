Animaciones con Keyframes: Experiencias interactivas sin JavaScript

1. La Anatomía de @keyframes

A diferencia de transition, que solo va de un punto A a un punto B, @keyframes permite definir múltiples paradas intermedias.

@keyframes flotar {
  0% { transform: translateY(0); opacity: 0; }
  50% { transform: translateY(-20px); opacity: 1; }
  100% { transform: translateY(0); opacity: 0.8; }
}

2. Propiedades Esenciales de animation

Se utiliza la propiedad abreviada para controlar el flujo de la animación:

  • name: Nombre del keyframe.
  • duration: Tiempo total (ej: 3s).
  • timing-function: Curva de aceleración (ease-in, linear).
  • iteration-count: Repeticiones (infinite o número).
  • fill-mode: Estado al terminar (forwards mantiene el final).

3. Animaciones Complejas: Encadenamiento

Puedes aplicar múltiples animaciones separándolas por comas para combinar efectos independientes.

.loader {
  animation: 
    girar 2s linear infinite,
    cambiarColor 5s ease alternate infinite;
}

4. Experiencias Interactivas

Interacción con Hover

Puedes pausar animaciones mediante animation-play-state: paused; al pasar el ratón.

El truco del Checkbox (Menús sin JS)

Usa el estado :checked de un input para disparar animaciones en otros elementos mediante el selector de hermano (~).

5. Rendimiento: El Secreto de las 60 FPS

Para evitar tirones (reflow), la regla de oro es animar solo propiedades que se ejecuten en la GPU:

  • transform: (translate, scale, rotate).
  • opacity: Para desvanecimientos.
