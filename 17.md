## 1. Google Fonts de Forma Profesional

Muchos desarrolladores simplemente copian el `<link>` de Google Fonts, pero esto añade peticiones externas que pueden ralentizar la carga.

### Mejores prácticas:

* **Limita los pesos:** No cargues *Light, Regular, Medium, Bold, Black* si solo vas a usar dos. Cada peso es un archivo extra.
* **Auto-hospedaje (Self-hosting):** Descarga las fuentes y sírvelas desde tu propio servidor. Esto elimina la resolución DNS de Google y permite usar HTTP/2 para una carga más rápida.
* **Preconectar:** Si usas los servidores de Google, añade esto para acelerar la conexión:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

```

## 2. Dominando `@font-face` y `font-display`

Cuando hospedas tus propias fuentes, usas la regla `@font-face` en CSS. La clave aquí es el formato **WOFF2**, que es el más comprimido y eficiente actualmente.

```css
@font-face {
  font-family: 'MiFuenteCustom';
  src: url('/fonts/mi-fuente.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap; /* CRÍTICO para el rendimiento */
}

```

### El secreto: `font-display: swap`

Esta propiedad le dice al navegador: *"Muestra una fuente del sistema (Arial, Roboto) mientras se descarga la fuente personalizada"*. Esto evita el **FOIT** (Flash of Invisible Text), donde el usuario no ve nada hasta que la fuente carga.

## 3. Unidades Relativas: `rem` vs `em`

Para un rendimiento web moderno y accesibilidad, **debes evitar los píxeles (`px`)** para el tamaño del texto.

* **`rem` (Root EM):** Se basa en el tamaño de fuente de la raíz (`<html>`). Por defecto, la mayoría de navegadores usan **16px**.
* *Uso:* Tamaños de fuente generales para mantener consistencia en todo el sitio.


* **`em`:** Se basa en el tamaño de fuente del elemento padre.
* *Uso:* Espaciados internos (padding/margin) o elementos que deben escalar proporcionalmente a su contenedor.



### La técnica del 62.5%

Para que el cálculo de `rem` sea más intuitivo (que 1rem = 10px), puedes usar este truco en tu CSS base:

```css
html {
  font-size: 62.5%; /* Cambia el base de 16px a 10px */
}

body {
  font-size: 1.6rem; /* Equivale a 16px, pero es escalable */
}

h1 {
  font-size: 3.2rem; /* Equivale a 32px */
}

```

## 4. Tipografía Fluida (Escalabilidad Total)

En lugar de usar muchos `media queries` para cambiar el tamaño del texto en móvil y desktop, puedes usar la función `clamp()`.

```css
h1 {
  /* clamp(mínimo, valor ideal, máximo) */
  font-size: clamp(2rem, 5vw + 1rem, 4rem);
}

```

Esto crea un título que crece y se encoge suavemente según el ancho de la ventana gráfica (**Viewport**).

## 5. Optimización de Recursos (Preload)

Si sabes que tu fuente principal es esencial para el renderizado, dile al navegador que empiece a descargarla de inmediato, incluso antes de que el CSS la solicite.

```html
<link rel="preload" href="/fonts/mi-fuente.woff2" as="font" type="font/woff2" crossorigin>

```

## Comparativa de Formatos y Unidades

| Concepto | Recomendación | Por qué |
| --- | --- | --- |
| **Formato** | WOFF2 | Mejor compresión (30% menos que WOFF). |
| **Unidad** | `rem` | Accesibilidad; permite al usuario escalar el texto. |
| **Carga** | `font-display: swap` | Evita que el usuario vea una página en blanco. |
| **Cantidad** | Máximo 2 familias | Cada fuente añade latencia y peso al sitio. |
