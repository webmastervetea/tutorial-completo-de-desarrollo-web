## 1. Formatos de Próxima Generación (WebP y AVIF)

Olvida el estándar de usar solo JPEG o PNG. Los formatos modernos ofrecen una compresión superior sin pérdida de calidad perceptible.

* **WebP:** Soporta transparencia (como PNG) y compresión eficiente (como JPEG). Es un 25-34% más ligero que un JPEG equivalente.
* **AVIF:** El nuevo estándar de oro. Ofrece hasta un 50% más de ahorro que JPEG, aunque su tiempo de codificación es mayor.

### Implementación con el elemento `<picture>`

Para asegurar la compatibilidad con navegadores antiguos, utiliza la etiqueta `<picture>`. El navegador elegirá el primer formato que reconozca.

```html
<picture>
  <source srcset="imagen.avif" type="image/avif">
  <source srcset="imagen.webp" type="image/webp">
  <img src="imagen.jpg" alt="Descripción de la imagen" width="800" height="600">
</picture>

```

## 2. Lazy Loading (Carga Diferida)

El **Lazy Loading** evita que el navegador descargue imágenes que el usuario aún no ve (las que están fuera de la pantalla inicial o "below the fold").

### Atributo nativo `loading="lazy"`

Hoy en día, no necesitas librerías pesadas de JavaScript. Los navegadores modernos lo soportan de forma nativa:

```html
<img src="foto.webp" loading="lazy" alt="Ejemplo de carga diferida">

```

**⚠️ Regla de oro:** Nunca apliques `loading="lazy"` a la imagen principal (LCP - Largest Contentful Paint). La primera imagen que ve el usuario debe cargar lo más rápido posible; para ella, usa `fetchpriority="high"`.

## 3. Atributos `srcset` y `sizes` (Imágenes Responsivas)

No envíes una imagen de 2000px a un teléfono móvil que solo tiene 400px de ancho de pantalla.

* **`srcset`**: Define una lista de archivos de diferentes tamaños.
* **`sizes`**: Indica al navegador qué tamaño de imagen debe elegir según el ancho de la pantalla.

```html
<img src="small.jpg" 
     srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1200w" 
     sizes="(max-width: 600px) 400px, 800px" 
     alt="Imagen adaptativa">

```

---

## 4. Prevención de CLS (Cumulative Layout Shift)

Uno de los errores más comunes es no definir dimensiones. Si no reservas el espacio, el contenido "salta" cuando la imagen termina de cargar, arruinando la experiencia del usuario.

**Solución:** Define siempre `width` y `height` en el HTML. El navegador usará estos valores para calcular la relación de aspecto (aspect-ratio) antes de que la imagen descargue.

```html
<img src="imagen.webp" width="800" height="600" alt="Evitando saltos de diseño">

```

## Resumen de Mejores Prácticas

| Técnica | Beneficio Principal |
| --- | --- |
| **Formato WebP/AVIF** | Reduce drásticamente el peso del archivo (KB). |
| **Lazy Loading** | Ahorra ancho de banda y mejora el tiempo de interacción. |
| **Srcset / Sizes** | Entrega el tamaño exacto necesario para el dispositivo. |
| **Dimensiones fijas** | Evita que el contenido se mueva mientras carga (CLS). |
