Optimización y Rendimiento: Fuentes Web y Tipografía Profesional

1. Google Fonts de Forma Profesional

Muchos desarrolladores simplemente copian el enlace de Google Fonts, pero esto añade peticiones externas que pueden ralentizar la carga.

Mejores prácticas:

  • Limita los pesos: No cargues pesos innecesarios; cada peso es un archivo extra.
  • Auto-hospedaje (Self-hosting): Sírvelas desde tu propio servidor para eliminar la resolución DNS externa y aprovechar HTTP/2.
  • Preconectar: Si usas sus servidores, acelera la conexión con preconnect:
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

2. Dominando @font-face y font-display

Al hospedar fuentes propias, utiliza el formato WOFF2, que es el más comprimido y eficiente actualmente.

@font-face {
  font-family: 'MiFuenteCustom';
  src: url('/fonts/mi-fuente.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap; /* CRÍTICO para el rendimiento */
}
El secreto: font-display: swap
Muestra una fuente del sistema mientras se descarga la personalizada, evitando el Flash of Invisible Text (FOIT).

3. Unidades Relativas: rem vs em

Para rendimiento y accesibilidad, evita los píxeles (px) para el tamaño del texto.

  • rem (Root EM): Se basa en el tamaño de fuente de la raíz (<html>), usualmente 16px por defecto.
  • em: Se basa en el tamaño de fuente del elemento padre, ideal para elementos que deben escalar proporcionalmente a su contenedor.

La técnica del 62.5%

Ajusta el base para que 1rem equivalga a 10px, facilitando el cálculo:

html { font-size: 62.5%; }
body { font-size: 1.6rem; } /* Equivale a 16px */

4. Tipografía Fluida

Usa clamp() para que el texto crezca y se encoja suavemente según el Viewport sin exceso de media queries.

h1 { font-size: clamp(2rem, 5vw + 1rem, 4rem); }

5. Optimización de Recursos (Preload)

Indica al navegador que descargue la fuente de inmediato si es esencial para el renderizado inicial:

<link rel="preload" href="/fonts/mi-fuente.woff2" as="font" type="font/woff2" crossorigin>

Comparativa de Formatos y Unidades

Concepto Recomendación Por qué
Formato WOFF2 30% menos peso que WOFF.
Unidad rem Accesibilidad y escalabilidad.
Carga font-display: swap Evita textos invisibles (FOIT).
Cantidad Máx. 2 familias Evita latencia y peso innecesario.
