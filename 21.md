## 1. La Anatomía de `@keyframes`

Mientras que `transition` solo permite ir de un punto A a un punto B, `@keyframes` te permite definir múltiples paradas intermedias.

```css
@keyframes flotar {
  0% {
    transform: translateY(0);
    opacity: 0;
  }
  50% {
    transform: translateY(-20px);
    opacity: 1;
  }
  100% {
    transform: translateY(0);
    opacity: 0.8;
  }
}

```

---

## 2. Propiedades Esenciales de `animation`

Para aplicar los frames definidos, usamos la propiedad abreviada `animation`. Es vital entender sus componentes:

1. **name:** El nombre del `@keyframes`.
2. **duration:** Cuánto tarda (ej: `3s`, `500ms`).
3. **timing-function:** La curva de aceleración (`ease-in`, `linear`, `cubic-bezier`).
4. **delay:** Tiempo de espera antes de empezar.
5. **iteration-count:** Cuántas veces se repite (`infinite` o un número).
6. **direction:** Si va hacia adelante (`normal`), atrás (`reverse`) o rebota (`alternate`).
7. **fill-mode:** Qué pasa al terminar (`forwards` mantiene el estado final).

```css
.card {
  animation: flotar 3s ease-in-out infinite alternate;
}

```

## 3. Animaciones Complejas: Encadenamiento

Puedes aplicar múltiples animaciones a un mismo elemento separándolas por comas. Esto es útil para combinar efectos independientes (como rotar y escalar al mismo tiempo).

```css
.loader {
  animation: 
    girar 2s linear infinite,
    cambiarColor 5s ease alternate infinite;
}

```

## 4. Experiencias Interactivas (Hover y Estados)

Las animaciones no tienen por qué ejecutarse solas. Podemos activarlas o pausarlas mediante interacciones del usuario.

### Pausar con el ratón

```css
.card:hover {
  animation-play-state: paused;
}

```

### El truco del Checkbox (Menús sin JS)

Podemos usar el estado `:checked` de un input oculto para disparar animaciones complejas en otros elementos usando el selector de hermano (`~`).

```css
/* Cuando el checkbox se marca, el menú se desliza */
#menu-toggle:checked ~ .side-menu {
  animation: slideIn 0.5s forwards;
}

```

## 5. Rendimiento: El Secreto de las 60 FPS

No todas las propiedades CSS son iguales. Si animas `width`, `height` o `top`, el navegador debe recalcular el diseño (**reflow**), lo que causa tirones.

**Regla de oro:** Para animaciones fluidas, usa solo:

* **`transform`:** (translate, scale, rotate, skew).
* **`opacity`:** Para desvanecimientos.

Estas propiedades se ejecutan en la GPU (aceleración por hardware) y no afectan al resto de los elementos de la página.

## Resumen de Mejores Prácticas

| Técnica | Recomendación | Propósito |
| --- | --- | --- |
| **`will-change`** | Úsala con moderación. | Avisa al navegador que un elemento se animará. |
| **`cubic-bezier`** | Personaliza tus curvas. | Crea movimientos más naturales y menos mecánicos. |
| **`prefers-reduced-motion`** | Respeta al usuario. | Desactiva animaciones si el usuario lo tiene configurado en su sistema. |
